Glossary
--------

.. dragndrop:: 02-dnd-3-glossary1
    :feedback: Look back in the chapter for these terms.
    :match_1: assignment|||A statement that assigns a value to a variable.
    :match_2: floating point|||A type that represents numbers with fractional parts.
    :match_3: integer|||A type that represents whole numbers.
    :match_4: keyword|||A reserved word that is used by the compiler to parse a program; you cannot use keywords like ``if``\ , ``def``\ , and ``while`` as variable names.
    :match_5: mnemonic|||A memory aid. We often give variables mnemonic names to help us remember what is stored in the variable. 
    :match_6: string|||A type that represents sequences of characters.
    :match_7: type|||A category of values such as integers (type ``int``\ ), floating-point numbers (type ``float``\ ), and strings (type ``str``\ ).
    :match_8: value|||One of the basic units of data, like a number or string, that a program manipulates.
    :match_9: variable|||A name that refers to a value.

    Match each term with its definition.

.. dragndrop:: 02-dnd-4-glossary2
    :feedback: Look back in the chapter for these terms.
    :match_1: concatenate|||To join two operands end to end.
    :match_2: comment|||Information in a program that is meant for other programmers (or anyone reading the source code) and has no effect on the execution of the program.
    :match_3: evaluate|||To simplify an expression by performing the operations in order to yield a single value.
    :match_4: expression|||A combination of variables, operators, and values that represents a single result value.
    :match_5: modulus operator|||An operator, denoted with a percent sign (\ ``%``\ ), that works on integers and yields the remainder when one number is divided by another.
    :match_6: operand|||One of the values on which an operator operates.
    :match_7: operator|||A special symbol that represents a simple computation like addition, multiplication, or string concatenation.
    :match_8: rules of precedence|||The set of rules governing the order in which expressions involving multiple operators and operands are evaluated.
    :match_9: statement|||A section of code that represents a command or action. So far, the statements we have seen are assignments and print expression statement.

    Match each term with its definition.
